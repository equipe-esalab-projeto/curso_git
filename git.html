Guia Completo de Comandos Git

Pré-requisitos e Configuração

git clone

Comando: git clone https://github.com/seu_id_de_usuario_github/fiplan

Simulação do Terminal:

Cloning into 'fiplan'...
remote: Enumerating objects: 10, done.
remote: Counting objects: 100% (10/10), done.
Receiving objects: 100% (10/10), 1.23 KiB | 1.23 MiB/s, done.
git config

Comando: git config --local user.name "Seu nome"

Simulação do Terminal:

(Silencioso - o comando não retorna nada em caso de sucesso)
git config --list --show-origin

Comando: git config --list --show-origin

Simulação do Terminal:

file:C:/Users/User/.gitconfig      user.name=Seu Nome
file:C:/Users/User/.gitconfig      user.email=seu_email@example.com
file:.git/config                   alias.lol=log --oneline --graph

Reset

Entenda o ciclo de vida dos arquivos no Git: WORKING DIRECTORY (diretório de trabalho) → STAGE (área de preparação) → REPO (repositório).

O comando git reset varia seu comportamento dependendo do modo utilizado (soft, mixed, hard).

git reset --soft

Comando: git reset --soft HEAD~3

Simulação do Terminal:

(Silencioso)
(O `git status` mostraria todos os arquivos dos três commits anteriores prontos para um novo commit)
git reset --mixed (padrão)

Comando: git reset --mixed HEAD~1

Simulação do Terminal:

Unstaged changes after reset:
M       UserVO
A       User
git reset --hard

Comando: git reset --hard HEAD~1

Simulação do Terminal:

HEAD is now at 6d702e5 docs(readme): Tem um erro aqui

Revert

O git revert desfaz um commit específico, criando um **novo commit** que reverte as alterações do commit original. É uma ação não destrutiva e segura para branches compartilhadas.

git revert

Comando: git revert commit_que_esta_errado

Simulação do Terminal:

[main a1b2c3d] Revert "docs(readme): Tem um erro aqui"
1 file changed, 10 insertions(+)

Cherry Pick

O git cherry-pick copia um ou mais commits de uma branch para outra, criando novos commits com as mesmas alterações.

git cherry-pick

Comando: git cherry-pick commit_hash

Simulação do Terminal:

[feature/nova_feature b1c2d3e] feat: Implementa novo componente
1 file changed, 25 insertions(+)
git cherry-pick --no-commit

Comando: git cherry-pick commit --no-commit

Simulação do Terminal:

(Silencioso. As alterações do commit são aplicadas no Working Directory, mas não em um novo commit)

Stash

O git stash é um recurso para salvar temporariamente as alterações não "commitadas", permitindo que você mude de branch ou realize outras tarefas sem perder o seu progresso.

git stash save

Comando: git stash save "descrição"

Simulação do Terminal:

Saved working directory and index state On main: "descrição"
git stash pop

Comando: git stash pop

Simulação do Terminal:

On branch main
Changes to be committed:
  (use "git restore --staged ..." to unstage)
    new file:   UserView

Dropped refs/stash@{0} (a1b2c3d...)

git add

O git add -p permite versionar mudanças (hunks) de forma seletiva, em vez de arquivos inteiros.

git add -p

Comando: git add -p

Simulação do Terminal:

diff --git a/README.md b/README.md
index 1a2b3c4..d1e2f3a 100644
--- a/README.md
+++ b/README.md
@@ -1,5 +1,6 @@
 # Project
+
-This is an old line.
+This is a new line.

Stage this hunk [y/n/a/d/e/?]? y

Log

O git log permite formatar e filtrar o histórico do versionamento, tornando a análise mais fácil.

git log --oneline --stat

Comando: git log --oneline --stat

Simulação do Terminal:

a1b2c3d (HEAD -> main) feat: adiciona nova funcionalidade
.../arquivo.js | 10 ++++++++++
1 file changed, 10 insertions(+)
git log --graph

Comando: git log --graph --oneline

Simulação do Terminal:

* a1b2c3d (HEAD -> main) feat: login
* d1e2f3a docs: Atualiza README
* c1d2e3f feat: homepage
git log com filtros

Comando: git log --author="preti" --oneline --grep="docs" -i

Simulação do Terminal:

c1d2e3f docs: corrige erro de digitação

CHANGELOG

A geração de changelogs serve para documentar as alterações entre as versões, mantendo desenvolvedores e usuários informados.

git-cliff

Comando: git cliff -o CHANGELOG.md

Simulação do Terminal:

[INFO] Found 1 new commits in the current branch.
[INFO] Generating changelog...
[INFO] Generated changelog saved to CHANGELOG.md.

Hooks

Hooks são scripts que o Git executa automaticamente antes ou depois de certos eventos, como commits e pushes.

pre-commit

Comando: git commit -m "commit com espaço desnecessário"

Simulação do Terminal:

(Se o hook `pre-commit` detectar espaços em branco)
README.md:10: trailing whitespace.
fatal: pre-commit hook failed
(O commit será abortado)

Outros Comandos Úteis

git push --force-with-lease

Comando: git push --force-with-lease

Simulação do Terminal:

To https://github.com/seu-usuario/seu-repo
+ d1e2f3a...a1b2c3d main -> main (forced update)
git branch --merged

Comando: git branch --merged main | grep -v '^\*' | xargs -n 1 git branch -d

Simulação do Terminal:

Deleted branch feature/login (was 1a2b3c4).
Deleted branch feature/dashboard (was d1e2f3a).
