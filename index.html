Tutorial de Comandos Git

Configuração Inicial

git config --global --list
Exibe todas as configurações globais do Git configuradas no sistema.
git config --global user.name "keren-hapuqueK"
Define o nome de usuário global para os commits do Git.
git config --global user.email "hapuque.o@estudante.ifmt.edu.br"
Define o email global para os commits do Git.
git init

git init= significa que você está inicializando um repositório Git vazio no diretório atual. Isso cria um novo repositório Git, permitindo que você comece a rastrear alterações nos arquivos desse diretório.

Diferença entre Repositório Local e Remoto

Repositório Local

  • Existe apenas no seu computador
  • Criado com o comando git init
  • Armazena todas as alterações que você faz em seus arquivos
  • Contém todo o histórico de commits feitos localmente
  • Você pode trabalhar offline, sem precisar de conexão com a internet

Repositório Remoto

  • Hospedado em um servidor na internet (como GitHub, GitLab, Bitbucket)
  • Permite colaboração entre diferentes desenvolvedores
  • Funciona como backup dos seus arquivos
  • Adicionado ao seu projeto com git remote add origin URL
  • Você envia alterações locais para o remoto com git push
  • Você baixa alterações do remoto com git pull

Fluxo de Trabalho Básico

  1. Trabalhe em arquivos no repositório local
  2. Adicione alterações à área de preparação (git add)
  3. Faça um commit das alterações (git commit)
  4. Envie os commits para o repositório remoto (git push)

git status
Mostra o estado atual do repositório Git, incluindo arquivos modificados, não rastreados ou preparados para commit.
keren_hapuque@KFOFIS:~/projeto_aulagit$ git status On branch master No commits yet Untracked files: (use "git add ..." to include in what will be committed) test.html nothing added to commit but untracked files present (use "git add" to track)
git add .
Adiciona todos os arquivos modificados à área de preparação (staging area) para serem incluídos no próximo commit.
keren_hapuque@KFOFIS:~/projeto_aulagit$ git add . keren_hapuque@KFOFIS:~/projeto_aulagit$ git status On branch master No commits yet Changes to be committed: (use "git rm --cached ..." to unstage) new file: test.html
git commit -m "test"
Cria um novo commit com todos os arquivos da área de preparação, usando "test" como mensagem de commit.
keren_hapuque@KFOFIS:~/projeto_aulagit$ git commit -m "test" [master (root-commit) adc3d1d] test 1 file changed, 3 insertions(+) create mode 100644 test.html keren_hapuque@KFOFIS:~/projeto_aulagit$ git status On branch master nothing to commit, working tree clean keren_hapuque@KFOFIS:~/projeto_aulagit$ git status On branch master nothing to commit, working tree clean

Conectando ao Repositório Remoto

git add origin https://github.com/Keren-HapuqueK/projeto_aulagit.git
Comando incorreto que resultou em erro. Para adicionar um repositório remoto, deve-se usar "git remote add" e não "git add".
keren_hapuque@KFOFIS:~/projeto_aulagit$ git add origin https://github.com/Keren-HapuqueK/projeto_aulagit.git fatal: pathspec 'origin' did not match any files
git remote add origin https://github.com/Keren-HapuqueK/projeto_aulagit.git
Adiciona um repositório remoto chamado "origin" com o URL especificado.
git branch -M main

git branch -M main: significa que mudou o nome da branch

Renomeia a branch atual (provavelmente "master") para "main", que é o novo padrão recomendado.
