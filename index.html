Tutorial de Comandos Git

Configuração Inicial

git config --global --list
Exibe todas as configurações globais do Git configuradas no sistema.
git config --global user.name "exemplo-usuario"
Define o nome de usuário global para os commits do Git.
git config --global user.email "usuario@exemplo.com"
Define o email global para os commits do Git.
git init

git init= significa que você está inicializando um repositório Git vazio no diretório atual. Isso cria um novo repositório Git, permitindo que você comece a rastrear alterações nos arquivos desse diretório.

Diferença entre Repositório Local e Remoto

Repositório Local

  • Existe apenas no seu computador
  • Criado com o comando git init
  • Armazena todas as alterações que você faz em seus arquivos
  • Contém todo o histórico de commits feitos localmente
  • Você pode trabalhar offline, sem precisar de conexão com a internet

Repositório Remoto

  • Hospedado em um servidor na internet (como GitHub, GitLab, Bitbucket)
  • Permite colaboração entre diferentes desenvolvedores
  • Funciona como backup dos seus arquivos
  • Adicionado ao seu projeto com git remote add origin URL
  • Você envia alterações locais para o remoto com git push
  • Você baixa alterações do remoto com git pull

Fluxo de Trabalho Básico

  1. Trabalhe em arquivos no repositório local
  2. Adicione alterações à área de preparação (git add)
  3. Faça um commit das alterações (git commit)
  4. Envie os commits para o repositório remoto (git push)

Fluxo de Trabalho Básico

usuario@EXEMPLO:~/projeto_git$ git config --global --list user.name=exemplo-usuario user.email=usuario@exemplo.com usuario@EXEMPLO:~/projeto_git$ git init hint: Using 'master' as the name for the initial branch. This default branch name hint: is subject to change. To configure the initial branch name to use in all hint: of your new repositories, which will suppress this warning, call: hint: hint: git config --global init.defaultBranch hint: hint: Names commonly chosen instead of 'master' are 'main', 'trunk' and hint: 'development'. The just-created branch can be renamed via this command: hint: hint: git branch -m Initialized empty Git repository in /home/usuario/projeto_git/.git/
git status
Mostra o estado atual do repositório Git, incluindo arquivos modificados, não rastreados ou preparados para commit.
usuario@EXEMPLO:~/projeto_git$ git status On branch master No commits yet Untracked files: (use "git add ..." to include in what will be committed) test.html nothing added to commit but untracked files present (use "git add" to track)
git add .
Adiciona todos os arquivos modificados à área de preparação (staging area) para serem incluídos no próximo commit.
usuario@EXEMPLO:~/projeto_git$ git add . usuario@EXEMPLO:~/projeto_git$ git status On branch master No commits yet Changes to be committed: (use "git rm --cached ..." to unstage) new file: test.html
git commit -m "test"
Cria um novo commit com todos os arquivos da área de preparação, usando "test" como mensagem de commit.
usuario@EXEMPLO:~/projeto_git$ git commit -m "test" [master (root-commit) adc3d1d] test 1 file changed, 3 insertions(+) create mode 100644 test.html usuario@EXEMPLO:~/projeto_git$ git status On branch master nothing to commit, working tree clean usuario@EXEMPLO:~/projeto_git$ git status On branch master nothing to commit, working tree clean

Conectando ao Repositório Remoto

git add origin https://github.com/exemplo-usuario/projeto_git.git
Comando incorreto que resultou em erro. Para adicionar um repositório remoto, deve-se usar "git remote add" e não "git add".
usuario@EXEMPLO:~/projeto_git$ git add origin https://github.com/exemplo-usuario/projeto_git.git fatal: pathspec 'origin' did not match any files
git remote add origin https://github.com/exemplo-usuario/projeto_git.git
Adiciona um repositório remoto chamado "origin" com o URL especificado.
git branch -M main

git branch -M main: significa que mudou o nome da branch

Renomeia a branch atual (provavelmente "master") para "main", que é o novo padrão recomendado.

Trabalhando com Branches

git branch
Lista todas as branches locais do repositório. A branch atual será destacada com um asterisco.
usuario@EXEMPLO:~/projeto_git$ git branch * main
git branch nova-feature
Cria uma nova branch chamada "nova-feature" a partir da branch atual, mas não muda para ela.
usuario@EXEMPLO:~/projeto_git$ git branch nova-feature usuario@EXEMPLO:~/projeto_git$ git branch * main nova-feature
git checkout nova-feature
Muda da branch atual para a branch "nova-feature".
usuario@EXEMPLO:~/projeto_git$ git checkout nova-feature Switched to branch 'nova-feature' usuario@EXEMPLO:~/projeto_git$ git branch main * nova-feature
git checkout -b outra-feature
Cria uma nova branch chamada "outra-feature" e imediatamente muda para ela. Combina os comandos "git branch" e "git checkout" em um único comando.
usuario@EXEMPLO:~/projeto_git$ git checkout -b outra-feature Switched to a new branch 'outra-feature' usuario@EXEMPLO:~/projeto_git$ git branch main nova-feature * outra-feature

Mesclando Branches (Merge)

git checkout main
Retorna para a branch principal (main) antes de realizar o merge.
usuario@EXEMPLO:~/projeto_git$ git checkout main Switched to branch 'main'
git merge nova-feature
Mescla as alterações da branch "nova-feature" na branch atual (main). Isto combina o histórico das duas branches.
usuario@EXEMPLO:~/projeto_git$ git merge nova-feature Updating adc3d1d..2f4e8b9 Fast-forward feature.html | 5 +++++ 1 file changed, 5 insertions(+)
git branch -d nova-feature
Deleta a branch "nova-feature" após ter mesclado suas alterações com sucesso. Use -D (maiúsculo) se quiser forçar a exclusão mesmo sem mesclar.
usuario@EXEMPLO:~/projeto_git$ git branch -d nova-feature Deleted branch nova-feature (was 2f4e8b9).

Resolvendo Conflitos de Merge

Conflitos ocorrem quando o mesmo trecho de código foi modificado de maneiras diferentes em duas branches. Quando isso acontece, o Git não consegue decidir automaticamente qual versão manter.

Exemplo de conflito:

keren_hapuque@KFOFIS:~/projeto_aulagit$ git merge outra-feature Auto-merging index.html CONFLICT (content): Merge conflict in index.html Automatic merge failed; fix conflicts and then commit the result.

Passos para resolver conflitos:

  1. Abra os arquivos com conflito em um editor
  2. Procure por marcações como <<<<<<<, ======= e >>>>>>>
  3. Edite o arquivo para manter o código desejado
  4. Remova as marcações de conflito
  5. Adicione o arquivo resolvido com git add
  6. Complete o merge com git commit

Outros Comandos Úteis

git log
Mostra o histórico de commits, incluindo autor, data e mensagem de commit.
git log --graph --oneline --all
Mostra uma visualização gráfica do histórico de branches e commits de forma condensada.
git stash
Guarda temporariamente alterações não commitadas para que você possa mudar de branch sem perder seu trabalho.
git stash pop
Recupera as alterações guardadas mais recentes e as aplica ao seu diretório de trabalho.
git reset --hard HEAD
Desfaz todas as alterações no diretório de trabalho e na área de preparação, retornando ao último commit. Use com cautela, pois perderá todas as alterações não commitadas.
git remote -v
Lista todos os repositórios remotos configurados para o repositório local, mostrando seus URLs.
git push origin main
Envia os commits da branch atual (main) para o repositório remoto chamado "origin".
git pull origin main
Baixa e mescla as alterações do repositório remoto "origin" na branch atual (main).
